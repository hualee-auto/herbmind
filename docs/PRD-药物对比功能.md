## 1. 需求背景

### 1.1 用户痛点

中医药学习中有大量**功效相似但应用不同**的药材，学习者经常混淆：

| 易混淆药对 | 相似点 | 差异点 |
|-----------|--------|--------|
| 麻黄 vs 桂枝 | 都发散风寒 | 麻黄发汗力强，桂枝温通经脉 |
| 当归 vs 川芎 | 都活血调经 | 当归偏补血，川芎偏行气 |
| 白芍 vs 赤芍 | 都归肝经 | 白芍养血柔肝，赤芍清热凉血 |
| 生地黄 vs 熟地黄 | 都滋阴 | 生地清热凉血，熟地补血滋阴 |

> **用户原话**: "我知道这两味药都能活血，但考试时总是分不清该用哪个..."

### 1.2 解决方案

提供**并排对比功能**，让用户直观看到两味药的差异，强化记忆。

---

## 2. 用户故事

### US-005-1 对比易混淆药材（主故事）

**作为** 中医学生，
**我希望** 并排对比两味相似药材的信息，
**以便** 清晰区分它们的功效差异和适用场景。

** acceptance criteria **:
- 可以同时查看两味药的完整信息
- 差异项有视觉高亮提示
- 支持快速切换对比对象

### US-005-2 从详情页进入对比

**作为** 应用用户，
**当我** 查看某味药的详情时，
**我希望** 看到推荐的易混淆药物并一键对比，
**以便** 及时澄清疑惑。

### US-005-3 从搜索选择对比

**作为** 应用用户，
**当我** 搜索到多味相关药材时，
**我希望** 选择其中两味进行对比，
**以便** 选出最适合的药材。

### 4.2 设计要点

1. **并排布局**：左右两栏，信息对齐，方便对比
2. **差异高亮**：不同项使用浅黄色背景高亮（✨标记）
3. **相同折叠**：完全相同的内容可折叠收起
4. **交换按钮**：顶部支持快速交换左右两药位置

## 5. 对比维度

### 5.1 基础信息对比

| 维度 | 说明 | 差异处理 |
|------|------|---------|
| 药名 | 中文名 + 拼音 | - |
| 图片 | 药材图片 | 并排展示 |
| 分类 | 功效分类 | 不同时高亮 |
| 性味 | 四气五味 | 不同时高亮 |
| 归经 | 归属经络 | 不同时高亮 |

### 5.2 核心内容对比

| 维度 | 说明 | 差异处理 |
|------|------|---------|
| 功效 | 功效列表 | 不同的功效高亮 |
| 主治 | 主治病症 | 不同的病症高亮 |
| 禁忌 | 禁忌事项 | 不同的禁忌高亮 |
| 用法 | 用法用量 | 不同时高亮 |

### 5.3 记忆辅助对比

| 维度 | 说明 | 差异处理 |
|------|------|---------|
| 记忆口诀 | 口诀对比 | 并排展示 |
| 考试要点 | 关键区别 | 重点突出 |

## 6. 数据需求

### 6.1 易混淆药对数据

需要在数据库中预设常见易混淆药对：

```json
{
  "similarPairs": [
    {
      "herbA": "danggui",
      "herbB": "chuanxiong",
      "reason": "都活血调经，但当归偏补血，川芎偏行气",
      "difficulty": "高频考点"

    {
      "herbA": "mahuang",
      "herbB": "guizhi",
      "reason": "都发散风寒，麻黄发汗力强，桂枝温通经脉",
      "difficulty": "极高频考点"

    {
```

### 6.2 数据字段

| 字段 | 类型 | 说明 |
|------|------|------|

| herbB | string | 第二味药ID |
| reason | string | 易混淆原因说明 |
| difficulty | string | 考试难度/频率 |

## 7. 交互设计

### 7.1 手势操作

| 手势 | 操作 | 说明 |
|------|------|------|
| 左右滑动 | 切换对比药 | 快速更换右侧药材 |

### 7.2 按钮操作

| 按钮 | 功能 | 位置 |
|------|------|------|
| 交换 | 交换左右两药位置 | 顶部导航栏 |
| 更换 | 更换右侧对比药 | 右侧药名下方 |
| 收藏 | 收藏当前对比组合 | 顶部导航栏 |
| 分享 | 分享对比结果 | 顶部导航栏（可选） |

## 8. 验收标准

### 8.1 功能验收

| 验收项 | 验收标准 | 优先级 |
|-------|---------|--------|
| 入口可见 | 详情页易混淆模块正常显示 | P0 |
| 对比跳转 | 点击对比按钮正确进入对比页 | P0 |
| 并排展示 | 两味药信息并排完整展示 | P0 |
| 信息完整 | 展示：药名、分类、性味、归经、功效、主治 | P0 |
| 差异高亮 | 不同项使用背景色高亮 | P1 |
| 交换功能 | 点击交换按钮左右互换 | P1 |
| 更换功能 | 支持更换右侧对比药 | P1 |
| 返回功能 | 返回按钮正常返回上一页 | P0 |

### 8.2 数据验收

| 验收项 | 验收标准 | 优先级 |
|-------|---------|--------|
| 预设药对 | 数据库包含至少 10 组常见易混淆药对 | P1 |
| 关联显示 | 详情页正确显示关联的易混淆药物 | P1 |
| 数据完整 | 每对药物都有对比原因说明 | P2 |

### 8.3 性能验收

| 验收项 | 验收标准 | 优先级 |
|-------|---------|--------|
| 加载速度 | 对比页打开时间 < 500ms | P1 |
| 切换流畅 | 更换对比药时无卡顿 | P1 |

### 8.4 兼容性验收

| 验收项 | 验收标准 | 优先级 |
|-------|---------|--------|
| 小屏适配 | 在小屏手机上正常显示 | P1 |
| 横屏支持 | 横屏时扩大对比区域 | P3 |
